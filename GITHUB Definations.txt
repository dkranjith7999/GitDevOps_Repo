git fetch:- only looks for updates from GitHub without changing your actual files, 

Git Pull:- while git pull downloads those updates and immediately forces them into your files

A pull request (PR) is a way to ask permission to add your code changes to a main project.

Git pull == Git fetch & git merge

Git Reset:-
Types in git reset Soft reset and Hard reset
Soft reset : It will keep all changes
Mixed : It will Keep the working copy Reset the Index
Hard : Discard all changes

(Erase History) The git reset command is used to undo changes by moving your current branch backward to a specific earlier commit.

Git Revert :- (Record the History) The git revert command is a safe "undo" button. Instead of deleting past mistakes, it creates a brand-new commit that does the exact opposite of a previous commit 

Git cherry-pick :- git cherry-pick can be useful for undoing changes. For example, say a commit is accidently made to the wrong branch. You can switch to the correct branch and cherry-pick the commit to where it should belong.

Git squash :- Git squash combines multiple commits into a single commit to simplify and clean up project history, making it more organised and easier to review.

Git Blame :- The git blame command shows you exactly who wrote each line of code in a file and when they changed it.

Git Rebase :- The git rebase command is a tool that takes your recent code changes (commits) and replays them on top of the latest version of another branch.

Git Stash :- The git stash command is like a "pause button" for your unfinished code. It lets you temporarily hide away your current work without making a permanent commit, giving you a clean slate to work on something else

git stash: Saves your uncommitted changes and cleans your workspace.git 
stash pop: Brings your saved changes back and deletes them from the temporary drawer.
git stash list: Shows you a list of all the different half-finished tasks you have hidden away.
